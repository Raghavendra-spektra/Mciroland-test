# Scenario 3 — Package Management

This scenario covers APT package administration on Ubuntu/Debian-based systems. You will install a web server package, pin it to prevent upgrades, and then perform a complete removal with cleanup.

---

## Task 1: Install and Hold a Package

> **Note:** Ensure the package name is entered exactly as specified to ensure validation works correctly.

1. Update the package repository metadata using `apt`.

2. Install the **nginx** web server package.

3. Verify the installed version of **nginx**.

4. Locate the **nginx** configuration directory.

5. Place **nginx** on hold to prevent it from being upgraded in future updates.

6. Confirm the hold is active.

---

## Task 2: Remove a Package Completely

> **Note:** Complete Task 4 before proceeding with this task.

1. Remove the hold on **nginx**.

2. Completely purge the **nginx** package along with its configuration files.

3. Remove any unused dependencies left behind by the package removal.

4. Verify that **nginx** no longer exists on the system.

5. Verify that the **nginx** configuration files no longer exist on the system.

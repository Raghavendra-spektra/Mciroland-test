# Scenario 5 — Static IP Configuration and Network Documentation

This scenario covers static network configuration using Netplan and network state documentation. You will assign a static IPv4 address, apply the configuration, and collect interface and routing information into a report file.

---

## Task 1: Configure a Static IP and Document the Network State

> **Note:** Back up the existing Netplan configuration before making any changes to ensure the original state can be restored if needed.

1. Back up the existing Netplan configuration file before making any changes.

2. Configure a static IPv4 address on the target interface with the following:
   - A static IPv4 address with subnet mask
   - A default gateway
   - DNS server addresses

3. Apply the new network configuration using `netplan apply`.

4. Collect the following information into **/tmp/network_report.txt**:
   - Interface details
   - Routing table entries
   - DNS resolution output

5. Verify that **/tmp/network_report.txt** contains all three of the following:
   - Interface information
   - Routing table entries
   - DNS resolution results

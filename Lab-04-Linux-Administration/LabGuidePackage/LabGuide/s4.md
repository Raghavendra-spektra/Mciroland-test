# Scenario 4 — Process Monitoring and Control

This scenario covers Linux process management. You will identify CPU-intensive processes, adjust scheduling priorities, and control a background process using POSIX signals.

---

## Task 1: Identify High CPU Processes and Adjust Priority

> **Note:** Follow the specified file paths and values exactly to ensure validation works correctly.

1. List the top processes sorted by CPU usage and save the output to **/tmp/cpu_report.txt**.

2. Identify the PID of the top CPU-consuming process and use `renice` to set its nice value to **10**.

3. Append the updated process priority details to **/tmp/cpu_report.txt** and verify the change.


---

## Task 2: Background Job and Signal Control

> **Note:** Record the PID after launching the background process, as it is required for subsequent steps.

1. Start the command `ping -i 1 127.0.0.1` in the background, redirecting its output to **/tmp/ping.log**.

2. Record the PID of the background process.

3. Send **SIGSTOP** to the process to pause it.

4. Confirm the process is in a stopped state.

5. Send **SIGCONT** to the process to resume it.

6. Send **SIGTERM** to gracefully terminate the process.

7. Verify the process is no longer running.

# Scenario 1 — User and Group Management

This scenario covers Linux user and group administration. You will create a structured group hierarchy for a DevOps department, assign users to appropriate groups, and configure account restrictions.

---

## Task 1: Create a Department Group Structure with Constrained User Access

> **Note:** Use exact names and IDs as specified to ensure validation works correctly.

1. Create a group named **devops** with GID **5001** and a second group named **devops-readonly**.

2. Create user **devadmin** with home directory `/home/devops/devadmin` and primary group as    **devops**, then set a maximum password age of **30 days** for this user.

3. Create user **devviewer** with home directory `/home/devops/devviewer` and primary group **devops-readonly**, then lock the account to prevent interactive logins.

---

## Task 2: Audit and Modify an Existing User Account

> **Note:** Use exact user name as specified to ensure validation works correctly.

1. Create a user with the user name **jsmith**.

2. Add **jsmith** to the **devops** group without removing any existing group memberships.

3. Change the default login shell for **jsmith** to `/bin/sh`.

4. Save the current group membership information for **jsmith** to **/tmp/jsmith_groups.txt**.
